Documentation

Set up Nohi, connect your store data, and extend with Skills.

Installation

Download the installer for your platform from the Downloads page.

  • macOS — Open the .dmg, drag Nohi to Applications, then open it. Allow it in System Settings → Privacy & Security if prompted.
  • Windows — Run the .exe installer and follow the prompts.
  • Linux — Make the .AppImage executable: chmod +x AgentNohi.AppImage && ./AgentNohi.AppImage

Configure API Key

On first launch, Nohi opens Settings. Choose your AI provider and paste your API key.

  • Anthropic — console.anthropic.com/keys
  • OpenAI — platform.openai.com/api-keys
  • Google — aistudio.google.com/apikey

Your API key and all store data is stored locally in ~/.nohi/. Nothing leaves your machine except AI inference requests.

First Chat

After saving your settings, click Chat in the sidebar. Try asking:

"Show me orders from the past 7 days"
"Which products have stock below 10 units?"
"Draft a flash sale email for customers inactive 60+ days"

Nohi shows each tool call it makes before executing — you always see exactly what it's doing.

Built-in Tools

Nohi has access to these tools by default:

read_file

Reads the contents of a file. Returns up to 10,000 characters.

{ "path": "/Users/you/notes.txt" }

write_file

Writes content to a file (creates or overwrites).

{ "path": "/Users/you/output.txt", "content": "Hello world" }

list_dir

Lists files and folders in a directory.

{ "path": "/Users/you/Desktop" }

run_bash

Runs a shell command and returns stdout/stderr. Dangerous patterns are blocked.

{ "command": "ls -la ~/Documents" }

Nohi Skills

Skills are Markdown files that inject context into the agent's system prompt when a user message matches a trigger pattern. Built-in e-commerce skills load automatically. Create custom skills to teach Nohi your store's specific context.

Where to put skills

  • Default location: ~/.nohi/skills/
  • Custom location: set in Settings → Skills Directory

Creating a skill

Create a .md file with YAML frontmatter:

~/.nohi/skills/flash-sale.md
---
name: flash-sale
description: Flash sale assistant
trigger: flash sale|promotion|discount|促销|折扣
---

When helping with flash sales:
- Always check current inventory before suggesting discount depth
- Recommend targeting customers inactive for 30–60 days
- Draft email subject lines with urgency: time-limited offers
- Suggest bundle deals for slow-moving SKUs

When you send a message containing words like "promotion" or "discount", this skill's content is added to the agent's context automatically.

Scheduled Tasks

Create recurring tasks that Nohi runs automatically on a schedule:

  • Daily sales digest at 8am
  • Low-stock alerts when inventory drops below a threshold
  • Weekly customer win-back campaign drafts

Configure in Settings → Scheduled Tasks, or by editing ~/.nohi/scheduled-tasks.json.

Supported Models

  • Anthropic — claude-sonnet-4-6, claude-opus-4-6, claude-haiku-4-5
  • OpenAI — gpt-4o, gpt-4o-mini, o1, o3-mini
  • Google — gemini-2.0-flash, gemini-2.5-pro

Skill File Format

---
name: <skill-name>
description: <one-line description>
trigger: word1|word2|word3
---

<instructions for the agent>
  • name — identifier, used in logs
  • description — human-readable summary
  • trigger — pipe-separated keywords; any match activates the skill

Config File

Settings are stored at ~/.nohi/config.json:

{
  "provider": "anthropic",
  "apiKey": "sk-ant-...",
  "model": "claude-sonnet-4-6",
  "skillsDir": "",
  "scheduledTasksEnabled": true
}

You can edit this file directly. Changes take effect on the next conversation.
